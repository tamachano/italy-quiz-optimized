##  体験設計

イタリア名所クイズを実際のユーザーの出会いがどのような印象を生むのか？
## 「ストーリー」で解列します。

まず、初めに目に飛び込んでくるのは

シンプルかつ情報量の少ないレイアウトですが、
モダンかつミニマルな印象を期待させることができます。

そして、思わずクリックしたくなるような文字でまずはスタートしてもらいます。

そして実際にクリックしますと、ここからはクイズ画面になり、

鮮やかなヨーロッパの風景が表示されることで
非日常的な体験を生み出します。


そして、画像と上部の余白にプログレスドットを実装することにより、

残り問題数を潜在意識に刷り込ませ、
あと問題はどれくらいあるのか？という情報の不確定性による心理的なストレスを軽減しています。

またヒントも実装しています。
絶対に必要な機能ではなく、そこにあることで、
これはエンタメであるという刷り込みと共に、生まれる安心感を実装しています。

そしてクイズの選択肢のhoverアクションはイタリアをイメージし、
深みのある緑にワインレッドのような赤い閃光と微小な光が走ることで高級感のある手触りを感じさせています。

このような細かい手触りの調整により、ストレスを軽減させることで、安心感を与えると同時に
高級感の演出とユーザーを丁寧に包み込む体験を生むための構造を設計しています。

そして、結果と同時にビジターへのメッセージを表示させることにより、
「行動へのフィードバック」が行われます。

この瞬間に生まれるポジティブな感情こそが体験の核であり、価値になると考えています。

結果画面では、「もう一度プレイする」ではなく、
「もう一度」というワードでリプレイボタンだとわかると判断し、「イタリアの景色に思いを馳せる」
というテイストにすることによりイタリアの世界観を引き立てています。

また、私の他開発のテーマであるイタリア旅行サイトAuroraへの動線を入れることで
イタリア旅行への疑似体験が広がるというストーリを設計しています。
